# Query：$ARGUMENTS

回答關於知識庫的問題：**$ARGUMENTS**

1. **先讀 index 與總論。** 讀 `wiki/index.md` 找出相關頁、讀 `wiki/overview.md` 掌握目前的整體判斷；必要時用 grep／搜尋補強。

2. **讀相關頁。** 鑽進相關的來源／實體／概念頁；需要時回溯到 `raw/` 原文確認細節。

3. **綜合作答。** 給出帶引用的答案 — 引用要指向具體 wiki 頁（`[[頁面]]`）或 `raw/` 來源。若證據不足、或頁面之間有矛盾，直說，不要硬湊。

4. **選擇輸出形式。** 視問題給最合適的形式：markdown 段落、比較表、清單；需要時可做圖（matplotlib）或投影片（Marp）。

5. **回存有價值的答案。** 如果這個答案本身就是新知識（一份比較、一段分析、一個發現的連結），問使用者要不要存成 `wiki/notes/<slug>.md`。同意就：建立該頁 → 登錄 `wiki/index.md` → 用 `[[wikilink]]` 連回相關頁 → 在 `wiki/log.md` 追加 `## [<今天日期>] query | <問題摘要>`。讓探索也能像來源一樣累積，而不是消失在對話裡。
