# Lint wiki $ARGUMENTS

對知識庫做一次健檢（若 `$ARGUMENTS` 有指定範圍，就只查那一塊）。掃描並回報：

1. **矛盾。** 不同頁對同一件事說法不一致 — 列出衝突頁與各自來源。
2. **過時主張。** 較新來源已推翻、但舊頁還沒更新的說法；一併檢查 `wiki/overview.md` 是否還反映目前所有頁的共識。
3. **孤兒頁。** 沒有任何其他頁連入的頁（用 grep 找 `[[頁名]]` 的反向連結）。
4. **缺頁。** 被反覆提到、卻還沒有自己頁面的重要實體／概念。
5. **缺連結。** 本該互連、卻沒連起來的頁。
6. **資料缺口。** 可以靠一次網路搜尋補上的明顯空白 — 提議要找的新來源。
7. **新問題。** 根據目前累積的內容，建議幾個值得用 `/query` 探究的方向。

**先回報發現清單讓使用者挑要修哪些，不要一次全改。** 修完後把修補動作追加到 `wiki/log.md`：`## [<今天日期>] lint | <範圍>`。
